# やまふじ農園 お米 注文予測

顧客のお米購入履歴から、次回購入時期を予測する社内向け Web アプリです。
さくらのレンタルサーバ Standard プラン以上で動作します。

- 言語: PHP 8.x（フレームワーク無し）
- DB: MySQL 5.7+ / 8.0
- 想定規模: 顧客 100 件まで
- 認証: なし（推測困難な URL を秘匿運用）

## カテゴリー

1. 業務用 (`business`) — 中央値 + 直近平均で高精度に予測
2. 常連 (`regular`) — 平均間隔で予測
3. 自由米 (`retail`) — 平均間隔で予測（履歴が少なくなりがちなため確信度は低めに表示）

## 1. さくらレンタルサーバでのセットアップ手順

### (1) MySQL データベースの作成

1. さくらのコントロールパネルにログイン
2. 「Webサイト/データ」 → 「データベース」 → 「新規追加」
3. データベース名を控える（例: `your_account_rice`）
4. データベースのユーザー名・パスワードを控える
5. 接続先ホスト名を控える（例: `mysql1234.db.sakura.ne.jp`）

### (2) `config.php` の編集

ローカルで `config.php` を開き、以下を編集します。

```php
const DB_HOST = 'mysql1234.db.sakura.ne.jp';
const DB_NAME = 'your_account_rice';
const DB_USER = 'your_account';
const DB_PASS = 'your_db_password';
```

### (3) ファイルのアップロード

FileZilla 等の FTP/SFTP クライアントで、さくらの `~/www/` 配下に **推測困難な名前のディレクトリ** を作って、本リポジトリのファイル一式をアップロードします。

```
~/www/rice-a8Xk29Lp/
├── index.php
├── config.php
├── .htaccess
├── robots.txt
├── README.md
├── lib/
├── views/
├── assets/
└── sql/
```

> ディレクトリ名 `rice-a8Xk29Lp` の部分は任意のランダム文字列にしてください。
> このディレクトリ名が **そのままアプリの URL** になります。
> 一度決めたら変えなくて OK です。

### (4) スキーマの適用

1. さくらのコントロールパネル → データベース → 「管理ツールログイン」(phpMyAdmin)
2. 該当 DB を選択 → 「SQL」タブ
3. `sql/schema.sql` の中身をコピペして実行

### (5) アクセス確認

ブラウザで以下にアクセス：

```
https://<アカウント>.sakura.ne.jp/rice-a8Xk29Lp/
```

ダッシュボード画面が表示されれば成功です。

### (6) スマホのホーム画面に追加（推奨）

iOS Safari や Android Chrome で「共有」→「ホーム画面に追加」しておくと、アプリのように起動できます。

## 2. 使い方

### 顧客を登録する
画面下の `◆ 顧客` → 右上の `+ 新規顧客` から登録します。
カテゴリー（業務用 / 常連 / 自由米）を必ず選んでください。

### 購入を記録する
画面下の `+ 記録` から、顧客・購入日・数量(kg) を入力します。
顧客詳細画面の `+ 購入を記録` ボタンからも可能です。

### 次回予測を確認する
- ダッシュボードに「1週間以内に注文予測」「1ヶ月以内」が表示されます。
- 顧客詳細画面で個別の予測日・確信度・平均間隔を確認できます。
- 履歴が **2件以上** 必要です。

### CSV を書き出す
画面下の `↓ データ` から、以下 2 種をダウンロードできます。

| 種類 | 用途 |
|---|---|
| 購入履歴 CSV | Excel 確認用 / バックアップ |
| 予測サマリー CSV | AI に貼り付けて分析させる用途 |

UTF-8 BOM 付きなので Excel でそのまま開けます。

### CSV を取り込む（既存履歴のインポート）
画面下の `↓ データ` → 「CSV から購入履歴を取り込む」から、Excel 等で管理していた過去の購入履歴を一括で取り込めます。

**CSV フォーマット（5 列固定）**

```csv
顧客名,カテゴリー,購入日時,数量(kg),メモ
山田商店,業務用,2026/01/15 10:00,30,
田中太郎,常連,2026/02/03,5,おまけサービス
佐藤花子,自由米,2026/03/10,10,
```

| 列 | 必須 | 説明 |
|---|---|---|
| 顧客名 | ✓ | 同名顧客が無ければ自動で新規作成 |
| カテゴリー | ✓ | `業務用` / `常連` / `自由米` のいずれか |
| 購入日時 | ✓ | `2026/01/15` または `2026/01/15 10:00`（時刻省略可）|
| 数量(kg) | ✓ | 数値（小数可） |
| メモ | | 任意 |

**動作**
- 1 行目のヘッダー行は自動でスキップ
- 文字コードは UTF-8 / Shift_JIS どちらでも自動判定
- 同じ顧客・同じ購入日時・同じ数量の行は **重複としてスキップ**（同じ CSV を 2 度取り込んでも履歴が二重にならない）
- エラー行は除外され、プレビュー画面で行番号と理由が表示される
- 取り込みはトランザクションで実行されるため、途中失敗時はすべてロールバック

## 3. セキュリティについて

- ログイン機能はありません。**URL を知っている人がアクセスできる前提** です。
- URL を SNS や検索エンジンにインデックスされないよう、ディレクトリ名を推測困難な文字列にしてください。
- `robots.txt` と `.htaccess` で検索エンジンクローラを拒否済みです。
- 万が一 URL が漏れた場合は、ディレクトリ名をリネームすれば即座に新 URL に変更できます（DB と中身はそのままで、フォルダ名のみ変更）。

### より厳しくしたい場合
`.htaccess` に以下を追記すると IP 制限ができます（例: 店舗の固定 IP のみ許可）。

```apache
<RequireAll>
    Require ip 203.0.113.10
    Require ip 198.51.100.0/24
</RequireAll>
```

## 4. プッシュ通知（PWA）

注文記録時に他の担当者の端末へプッシュ通知を送る機能です。

### セットアップ手順（管理者・初回のみ）

1. **DB マイグレーション**
   既に運用中の場合は、phpMyAdmin で `sql/schema_push.sql` を実行してください。新規構築なら `sql/schema.sql` だけで OK です。

2. **VAPID 鍵の生成**
   ブラウザで `https://<アプリの URL>/index.php?p=vapid_setup` にアクセスし、「VAPID 鍵を生成する」ボタンを押す。表示された 3 行を `config.php` に貼り付け、保存して FTP アップロード。

3. **担当者を登録**
   画面下「データ」→「担当者マスタ」から、注文を記録する人（あなた、奥様 など）を追加。

### 各端末でのセットアップ（担当者ごと）

1. **PWA としてインストール**（必須）
   - Safari でアプリを開く
   - 共有ボタン →「ホーム画面に追加」
   - ホーム画面のアイコンから起動する

2. **通知の有効化**
   - 「データ」→「この端末の通知設定」を開く
   - 「私は誰？」で自分の担当者名を選択
   - 「この端末で通知を有効にする」をタップ → 通知許可ダイアログを許可
   - 「テスト通知を送る」で動作確認

### 通知の挙動

- 購入記録時、**記録者本人を除く全担当者の端末**にプッシュ通知が届きます
- 通知文の例: 「🌾 注文記録: 山田商店 / 田中さんが記録 / 業務用 / 30 kg / 2026/04/27 10:30」
- 通知タップで該当顧客の詳細画面が開きます
- 端末を長期間（数週間）使わないと購読が失効することがあります → 設定画面から再有効化してください

### iOS の制約（重要）

- iOS 16.4 以降が必須（iPhone 11/12 mini なら問題なし）
- **必ず「ホーム画面に追加」した PWA から開いた状態でないと通知が届きません**
- Safari のタブで開いている間は通知を受け取れません

---

## 5. データのバックアップ

- 月 1 回程度、`予測サマリー CSV` または `購入履歴 CSV` をダウンロードして保存しておくと安心です。
- DB 自体のバックアップは、phpMyAdmin の「エクスポート」機能でも取得できます。
